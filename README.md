# Software Engineering Group Project

This repository contains a university group software engineering project developed as part of a Software Engineering module.

The project is a Java Spring Boot MVC web application focused on gamified learning, user progress tracking, and performance recording. It was originally developed and managed on GitLab using Agile Scrum workflows, issue boards, weighted user stories, sprint planning, merge requests, testing documentation, and approval stages.

The purpose of this repository is to document the project and demonstrate practical experience with Java, Spring Boot MVC, SQL-backed data persistence, MVC architecture, Agile Scrum, Git-based version control, and collaborative software development.

---

## Project Overview

The application was designed to allow users to interact with learning/progress-based features while their activity and performance were tracked through gamified systems.

The main idea behind the project was to make user progress more visible and engaging by using features such as XP, levels, streaks, leaderboards, saved progress, start-time records, completion records, and search/filter functionality.

The project followed a typical Spring Boot MVC structure, using controllers, services, models, templates, and database-backed application logic.

## Database Usage

SQL was used during the project setup and development to support persistent data storage.

The database was used to store and retrieve application data linked to user progress and gamification features. This likely included data such as user records, XP, levels, daily streaks, saved progress, start times, completion times, fastest completion times, and leaderboard-related information.

Using SQL allowed the application to maintain user progress across sessions instead of relying only on temporary in-memory data.

---

## Application Architecture

The project followed a Spring Boot MVC structure.

The application was organised around:

* Controllers to handle web requests and route users to the correct pages
* Services to manage business logic and application behaviour
* Models/entities to represent application data
* Templates/views to display pages to users
* Database logic to store and retrieve persistent application information

This structure helped separate responsibilities within the application and made the codebase easier to manage as a group project.

## Main User Stories

Some of the main user stories/features included:

### Daily Streak

The daily streak feature tracked consecutive user activity and encouraged users to return to the application regularly.

### XP System

The XP system allowed users to gain experience points based on progress or completed activities.

### Level System

The level system converted XP progress into visible user levels, helping users understand their progress more clearly.

### Leaderboard

The leaderboard feature allowed users to compare performance or progress with others.

### Save and Load Progress

The save/load progress feature allowed user progress to be stored and retrieved, making the application more persistent and user-friendly.

### Record Start Times

This feature recorded when users started activities or tasks.

### Record Completion

This feature recorded when users completed activities or tasks, supporting performance tracking.

### Fastest Completion Times

The application could track completion times and display the fastest results linked to user performance.

### Search and Filter Results

The search/filter feature allowed users to search through available results, categories, or records more easily.
